# v0.6 / v0.7-rc.0 → v0.7-rc.1 마이그레이션 가이드

폴라리스 디자인 시스템 **v0.7.0-rc.1**은 디자인팀의 "Polaris Office Design System v1 (2026.05) — DESIGN.md + primitive-color-palette" 정의서에 완전 정렬한 release candidate입니다. 핵심은 다섯 가지:

1. 컬러 primitive 확장 — step `05`/`90` 추가, 5개 신규 패밀리(Sky Blue/Blue/Violet/Cyan/Yellow)
2. 시맨틱 토큰 19개 신설 — `--label-disabled`, `--layer-surface/-overlay`, `--accent-action-*` (Black 버튼) 등 + 다크모드 그레이스케일 재작성
3. Radius 스케일 한 단계 시프트 — `md` 8→12, `lg` 12→16, 기본값 `md` (12px)
4. Typography 11레벨 spec 명명 — `display`/`title`/`heading1-4`/`body1-3`/`caption1-2` + 18/13/11px 추가 + caption weight 700
5. Spacing/Z-index/Motion/Breakpoint 토큰 시스템 신설
6. Button 6 사이즈 (24/32/40/48/54/64) + Black variant
7. Input 52px + floating title, Modal 24px radius, Toast dark-blur 48px

v0.6, v0.7-rc.0 토큰 이름은 **deprecated alias로 rc.1에서 계속 작동**합니다. 자동화 codemod로 한 번에 옮길 수 있고, 옮기지 않아도 빌드는 깨지지 않습니다.

```bash

```

처리 항목:
- TS/TSX 토큰 멤버 접근: `text.primary` → `label.normal`, `surface.raised` → `layer.surface`, `brand.primary` → `accentBrand.normal`, `primary.normal` (rc.0) → `accentBrand.normal` 등
- 타이포그래피 키: `displayLg` / `h1` / `headingLg` → spec 이름 (`display` / `display` / `heading2`)
- Tailwind 클래스: `text-fg-primary` → `text-label-normal`, `bg-surface-raised` → `bg-layer-surface`, `text-polaris-h1` → `text-polaris-display`, `text-polaris-meta` → `text-polaris-caption1`, `bg-blue-5` → `bg-blue-05` 등
- CSS 커스텀 속성: `--polaris-text-primary` → `--polaris-label-normal`, `--polaris-primary-normal` → `--polaris-accent-brand-normal`
- `rounded-polaris-full` → `rounded-polaris-pill`

codemod가 처리하지 못하는 케이스 (수동 검토 필요):
- 동적 클래스 조립 (`\`text-${tone}-primary\``)
- Button `size` prop 변경 — 사이즈 의미가 시프트됐으므로 코드 검토 필요 (아래 §3.6 참조)
- 컴포넌트 prop으로 전달되는 토큰명
- Storybook MDX 인라인 hex

## 3. 시각적 변화 — 수동 확인 필요

### 3.1 브랜드 컬러 hex (rc.0과 동일)

|---|---|---|
| `--accent-brand-normal` (PO Blue) | `#2B7FFF` | `#1D7FF9` |
| AI Purple | `#7C5CFF` | `#6F3AD0` |
| Sheet Green | `#1FAE53` | `#51B41B` |
| Slide Orange | `#F37021` | `#FD8900` |
| PDF Red | `#E5413A` | `#F95C5C` |

### 3.2 컬러 primitive 확장 (rc.1 추가)

각 브랜드 램프에 step `90` (가장 어두운) 추가, step `5` → `05` 표기 변경. 5개 신규 패밀리 추가:

```
bg-sky-blue-50      // #2BAFF4 (plan badges)
bg-blue-supplementary-50  // #4C5BE5 (links, supplementary)
bg-violet-50        // #5C47C6 (charts, NOT for AI)
bg-cyan-50          // #00BADB (image/media format)
bg-yellow-50        // #F2B50B (note/highlight; NOT for warning)
```

`5` 표기는 deprecated alias로 계속 작동.

### 3.3 시맨틱 토큰 명명 변경

가장 큰 변경. 19개 신규 토큰이 추가되고 일부 rc.0 이름이 spec 이름으로 옮겨갑니다.

| `text.muted` | `label.alternative` |

| (없음) | `label.disabled` (NEW) |
| `surface.canvas` | `background.base` |
| `surface.raised` | `layer.surface` (NEW 분리) |
| `surface.sunken` | `fill.neutral` (NEW) |
| `surface.border` | `line.neutral` |
| `surface.borderStrong` | `line.normal` |
| (없음) | `line.strong` (NEW) |
| (없음) | `line.disabled` (NEW) |
| `interaction.hover` | `interaction.hover` |
| (없음) | `interaction.pressed` (NEW) |
| `fill.normal` | `fill.normal` |
| (없음) | `fill.strong` (NEW) |
| `brand.primary` / rc.0 `primary.normal` | `accentBrand.normal` |
| `brand.primaryHover` / rc.0 `primary.strong` | `accentBrand.strong` |

### 3.4 Radius 스케일 한 단계 시프트

| 키 | rc.0 | rc.1 |
|---|---|---|
| `xs` | 4 | 4 ✓ |
| `sm` | 6 | **8** |
| `md` | 8 | **12** ★ default |
| `lg` | 12 | **16** |
| `xl` | 16 | **24** |
| `2xl` | 24 | **38** |
| `pill` | 9999 | 9999 ✓ |

모든 컴포넌트가 한 단계 더 둥글어집니다 (Card, Modal, Button, Input). 컴포넌트 모서리 검수 필요.

### 3.5 타이포그래피 spec 명명

11레벨로 재정렬. `display`(60→40), Caption weight 400→700, body letter-spacing 제거.

| rc.0 | rc.1 (spec) |
|---|---|
| `display` (60) | **`display` (40)** ← 사이즈 축소 |
| `h1` (40) | `display` (40) — 같은 사이즈 |
| `h2` (32) | `title` (32) |
| `h3` (28) | `heading1` (28) |
| `h4` (24) | `heading2` (24) |
| `h5` (20) | `heading3` (20) |
| (없음) | `heading4` (18) **NEW** |
| `body` (16) | `body1` (16) |
| `bodySm` (14) | `body2` (14) |
| `detail` (14/M) | (계속 사용 가능 — spec엔 직접 매핑 없음) |
| (없음) | `body3` (13) **NEW** |
| `meta` (12/400) | `caption1` (12/**700**) ← weight 변경 |
| `tiny` (10/400) | `caption2` (11/**700**) ← 사이즈+weight 변경 |

모바일 (≤767px): 자동으로 한 단계씩 축소 (`tokens.css`의 `@media` 블록).

### 3.6 Button 6 사이즈 + Black variant

| rc.0 | 사이즈 | rc.1 | 사이즈 |
|---|---|---|---|
| `sm` | 26 | `xs` | 24 (-2) |
| `md` | 32 | `sm` | 32 ✓ |
| `lg` | 40 | `md` | 40 ✓ |
| (없음) | — | `lg` | 48 **NEW** |
| (없음) | — | `xl` | 54 **Large CTA** |
| (없음) | — | `2xl` | 64 **Hero CTA** |

**rc.0 → rc.1 사이즈 매핑** (수동 검토 필요):
- `<Button size="sm">` (rc.0 26h) → `<Button size="xs">` (rc.1 24h, -2px)
- `<Button size="md">` (rc.0 32h) → `<Button size="sm">` (rc.1 32h, 동일)
- `<Button size="lg">` (rc.0 40h) → `<Button size="md">` (rc.1 40h, 동일)

⚠️ codemod는 size prop을 자동 변환하지 않습니다 (다른 컴포넌트의 size prop과 구분 불가). 수동 변경하거나 rc.0 default size="md"를 그대로 두면 rc.1에서 40px로 렌더링됩니다.

신규 variant `dark` (Black 버튼) 사용 가능:
```tsx
<Button variant="dark">완료</Button>  // 검정 배경 흰 글자, 다크모드에서 자동 반전
```

`outline` variant는 `tertiary`로 rename됨 (alias 유지).

### 3.7 Input 52px + Floating Title

`<Input>` 높이가 36 → **52px**로 커지고, label이 입력 영역 안에 들어가는 floating title 패턴으로 바뀝니다.

```tsx
<Input
  label="이름"
  placeholder="홍길동"
  hint="2~10글자"
/>
```

- 빈 + 비포커스: label이 입력 영역 중앙 (placeholder 위치)
- 포커스 또는 값 있음: label이 12px / Bold로 상단 이동
- error 시: 좌측 ⚠️ 아이콘 + state-error 텍스트 자동 추가 (WCAG 1.4.1)

### 3.8 Modal radius 24px

`<Dialog>`가 `--layer-surface` 배경 + 24px radius (`--radius-xl`)로 바뀝니다. rc.0의 12px에서 두 배 더 둥글어짐. 보더 제거 (radius + shadow + tonal step만으로 깊이감 표현 — spec 철학).

### 3.9 Toast dark + blur

모든 Toast가 spec의 통일된 다크 글래스 표면으로 바뀝니다:
- `bg-layer-overlay backdrop-blur-md`
- 흰 텍스트 (`text-static-white`)
- min-h-12 (48px), 12px radius
- Variant는 left border 색상만 변경 (success 녹색, error 빨강 등)

rc.0의 파스텔 배경(`bg-status-success/10` 등)은 사라집니다. 알림 시각적 톤이 더 차분해집니다.

## 4. 새 토큰 시스템 4개

```ts
// Spacing (named)
import { spacingNamed } from '@polaris/ui';  // 4xs (2) → 4xl (64)

// Z-index

className="ease-polaris-out"                 // cubic-bezier(0, 0, 0.2, 1)

// Breakpoint (semantic alongside Tailwind defaults)
import { breakpoint } from '@polaris/ui';
breakpoint['tablet-h']  // '1024px'
```

## 5. 신규 lint 룰

**`@polaris/state-color-with-icon`** (warn): `text-state-success/-warning/-error` 가 아이콘 동반 없이 사용되면 경고. WCAG 1.4.1 (정보를 색상만으로 전달하지 않음) 준수 보조.

```tsx
{/* ❌ warns */}
<p className="text-state-error">잘못된 형식입니다</p>

{/* ✓ */}
<p className="text-state-error flex gap-polaris-3xs">
  <AlertCircle className="h-4 w-4" /> 잘못된 형식입니다
</p>
```

수동 false positive 회피: `// eslint-disable-next-line @polaris/state-color-with-icon`.

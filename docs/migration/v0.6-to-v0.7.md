# v0.6 → v0.7 마이그레이션 가이드

폴라리스 디자인 시스템 v0.7.0은 디자인팀의 "Polaris Office Design System v1 (2026.05)" 정식 정의서에 맞춰 토큰 명명·값·컴포넌트 스펙을 재정렬한 **breaking 릴리즈**입니다.

핵심 변경:
1. 브랜드 컬러 hex 갱신 (PO Blue / Sheet Green / Slide Orange / PDF Red / AI Purple)
2. 9-step 컬러 램프 신규 + 8-level radius 스케일 + AI Purple shadow
3. 시맨틱 토큰 명명 변경: `text.*` → `label.*`, `surface.*` → `background.*`/`line.*`, `brand.secondary*` → `ai.*` 등
4. 타이포그래피 H1–H5 + weight 700 + Detail/Meta/Tiny 신설
5. Button/Input/NovaInput 스펙 재정렬 (높이·radius·focus 효과)

v0.6 토큰 이름은 **deprecated alias로 v0.7에서 계속 작동**합니다. 자동화된 codemod로 한 번에 새 이름으로 옮길 수 있고, 옮기지 않아도 빌드는 깨지지 않습니다. v0.8에서 alias가 제거되니 미루지는 마세요.

---


이걸로 대부분 끝납니다. 나머지 visual diff는 아래 섹션에서 확인하세요.

## 1. 의존성 업데이트

```bash
pnpm up @polaris/ui @polaris/lint @polaris/plugin --latest
# 또는 package.json에서 ^0.7.0으로 직접 변경 후
pnpm install
```

`create-polaris-app`도 v0.7로 새 프로젝트를 생성합니다.

---

## 2. 자동 codemod 실행

대부분의 변경은 codemod가 처리합니다.

```bash
# 미리 확인 (변경하지 않음)
pnpm dlx @polaris/lint polaris-codemod-v07 src

# 실제 적용
pnpm dlx @polaris/lint polaris-codemod-v07 --apply src
```

처리 항목:
- TS/TSX 코드의 토큰 import: `text.primary` → `label.normal` 등
- Tailwind 클래스: `text-fg-primary` → `text-label-normal`, `text-polaris-heading-lg` → `text-polaris-h4`
- CSS 커스텀 속성: `--polaris-text-primary` → `--polaris-label-normal`
- `rounded-polaris-full` → `rounded-polaris-pill`

codemod가 처리하지 못하는 케이스 (수동 검토 필요):
- 동적 클래스 조립 (`\`text-${tone}-primary\``)
- 컴포넌트 prop으로 전달되는 토큰명
- Storybook MDX의 인라인 hex 값

---

## 3. 시각적 변화 — 수동 확인 필요

코드는 컴파일되지만 화면이 달라지는 항목입니다. PR 머지 전 visual regression 테스트를 돌리거나 핵심 화면을 직접 확인하세요.

### 3.1 브랜드 컬러 hex 갱신

| 토큰 | v0.6 | v0.7 |
|---|---|---|
| `brand.primary` (PO Blue) | `#2B7FFF` | `#1D7FF9` |
| `brand.secondary` (AI Purple) | `#7C5CFF` | `#6F3AD0` |
| Sheet Green | `#1FAE53` | `#51B41B` |
| Slide Orange | `#F37021` | `#FD8900` |
| PDF Red | `#E5413A` | `#F95C5C` |

각 hex 차이는 작아 보여도 브랜드 일관성에 영향을 줍니다. 마케팅 페이지 hero 영역, 로고 옆 액센트 등 brand-color에 의존하는 영역을 확인하세요.

### 3.2 타이포그래피 — heading weight 600 → 700

`headingLg` (H4), `headingMd` (H5)는 weight 600 (SemiBold)에서 **700 (Bold)** 로 바뀝니다. 화면이 살짝 무거워질 수 있으니 다이얼로그 타이틀, 카드 헤더 등을 점검하세요.

`displayLg`도 48px → 60px로 커집니다.

신설 토큰: `display`, `h1`, `h2`, `h3`, `h4`, `h5`, `body`, `body-sm`, `detail` (14/Medium), `meta` (12), `tiny` (10).

### 3.3 Button — 사이즈 한 단계씩 축소

| 사이즈 | v0.6 (height) | v0.7 (height) |
|---|---|---|
| `lg` | 48px | 40px |
| `md` | 40px | 32px |
| `sm` | 32px | 26px |

폼 영역, 카드 액션, 다이얼로그 footer 등에서 버튼이 작아 보일 수 있습니다. 시각적 균형이 깨지면 한 단계 위 사이즈로 올리세요 (예: `md` → `lg`).

신규 variant `ai`도 사용 가능 — AI 액션용 버튼에 `<Button variant="ai">`.

### 3.4 Input — 36px height + 3px focus glow

`<Input>` 높이가 40 → 36px로 줄어듭니다. focus 시 솔리드 2px ring 대신 1px PO Blue border + 3px 25% alpha glow가 적용됩니다. 폼이 더 가벼워 보입니다.

### 3.5 NovaInput — 단일행 → 두 줄 컴포저

NovaInput의 외형이 크게 바뀝니다.

- **v0.6**: 알약 모양 한 줄, 입력칸 + 우측 send 버튼.
- **v0.7**: 12px radius 사각 컴포저, 상단=입력, 하단=모델 pill(`modelPill` prop) + send 버튼. AI Purple glow shadow.

기존 코드는 컴파일되지만 모양이 다릅니다. 새 prop `modelPill`로 모델 셀렉터를 추가할 수 있습니다.

```tsx
<NovaInput
  placeholder="무엇이든 물어보세요."
  modelPill={<span>✦ Polaris GPT-4</span>}
  onSubmit={handleSubmit}
/>
```

### 3.6 Radius 스케일 — md 10→8, lg 14→12

기존 컴포넌트가 약간 덜 둥글어 보입니다. 카드/모달 코너를 점검하세요. 신규 단계: `2xs` (2), `xs` (4), `2xl` (24), `pill` (9999, `full`의 alias).

---

## 4. Deprecated alias — 언제 제거되나

v0.7는 **모든 v0.6 토큰명을 alias로 유지**합니다. v0.7 출시 직후 codemod를 돌리지 않아도 코드가 깨지지 않습니다.

- v0.7.x: alias 작동 (codemod 권장)
- v0.8.0: alias 제거. 그때까지 codemod를 돌려야 합니다.

ESLint deprecation warning은 v0.7.1에서 추가됩니다.

---

## 5. CI 점검

`@polaris/lint`의 `no-hardcoded-color` / `no-arbitrary-tailwind` 규칙은 v0.7에서도 동일하게 작동합니다. 새 토큰을 사용하면 자동으로 통과합니다.

기존 lint 우회용 `eslint-disable-next-line` 코멘트가 있다면 codemod 적용 후 다시 확인 — 새 토큰으로 충분히 표현 가능한 경우가 많습니다.

---

## 6. 문제 해결

### "Property 'displayLg' does not exist"
`@polaris/ui`를 0.7.0으로 정상 업그레이드했는지 확인. alias가 살아있어야 정상.

### "Cannot find name 'rounded-polaris-pill'"
Tailwind 캐시 클리어: `rm -rf .next node_modules/.cache && pnpm install`.

### Visual regression 베이스라인 깨짐
`pnpm test:visual --update-snapshots`로 새 베이스라인을 생성하세요. v0.7 변경은 의도된 시각 변경입니다.

### 더 큰 문제
사내 #design-system 채널에 문의하거나 [GitHub issue](https://github.com/miles-hs-lee/PolarisDesign/issues)를 등록하세요.

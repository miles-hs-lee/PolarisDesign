# v0.7 → v0.8 마이그레이션 가이드

폴라리스 디자인 시스템 **v0.8.0** 은 v0.7에서 누적된 alias / 미세 비일관성을 한 번에 정리한 BREAKING 릴리스입니다. 사내 컨슈머가 아직 소수일 때 구조적 변경을 끝내는 것이 목표.

핵심은 **단 하나의 codemod로 끝남** — `polaris-codemod-v08`이 토큰·Tailwind·CSS 변수·JSX prop·HStack/VStack 컴포넌트 이름까지 모두 한 번에 변환합니다.

> v0.7-rc.0 / v0.7-rc.1 / v0.7.x → v0.8 모두 동일 codemod로 인식. v0.6 이전이라면 먼저 [v0.6 → v0.7 가이드](v0.6-to-v0.7.md)를 진행한 뒤 v0.8로 옮기세요.

---

## 한 줄 요약

```bash
pnpm up @polaris/ui @polaris/lint @polaris/plugin --latest
pnpm dlx @polaris/lint polaris-codemod-v08 --apply src
pnpm build && pnpm lint
```

> **codemod 적용 범위**: 소비자 앱 코드(`src/`, `apps/`, `app/`)에만 적용. `@polaris/ui` 자체 소스(특히 `packages/ui/src/{tokens,styles,tailwind}`)는 마이그레이션 안내 docstring에 옛 alias 이름이 의도적으로 들어 있어 false-positive를 보입니다 — 제외하세요.

## 3. 무엇이 바뀌었나 — 항목별 매핑

### 3.1 토큰 / Tailwind 클래스 / CSS 변수 (Alias 청소)

v0.7 동안 `--all-aliases` 모드로 emit되어 작동하던 v0.6 / rc.0 / rc.1 이름들을 v0.8 빌드에서 모두 제거. **codemod 미적용 코드는 dead-utility(스타일 미적용)** 가 됩니다.

#### 라벨 / 텍스트

| v0.7 alias | v0.8 spec |
|---|---|
| `text-fg-primary` | `text-label-normal` |
| `text-fg-secondary` | `text-label-neutral` |
| `text-fg-muted` | `text-label-alternative` |
| `text-fg-on-brand` / `-on-status` | `text-label-inverse` |
| TS: `text.primary/.secondary/.muted/.onBrand/.onStatus` | `label.normal/.neutral/.alternative/.inverse/.inverse` |
| CSS: `var(--polaris-text-primary)` 등 | `var(--polaris-label-normal)` 등 |

#### 표면 / 보더 (Surface 패밀리 분할)

| v0.7 alias | v0.8 spec |
|---|---|
| `bg-surface-canvas` | `bg-background-base` |
| `bg-surface-raised` | `bg-layer-surface` |
| `bg-surface-sunken` | `bg-fill-neutral` |
| `border-surface-border` | `border-line-neutral` |
| `border-surface-border-strong` | `border-line-normal` |
| `bg-background-normal` | `bg-background-base` |
| `bg-background-alternative` | `bg-fill-neutral` |

> `bg-surface-popover` / `bg-surface-modal` (v0.7.5 elevation tier)은 **유지** — alias가 아니라 정상 토큰입니다. 다크모드에서 modal-on-popover 같은 layered elevation에 그대로 사용.

#### 브랜드 / AI / 액션

| v0.7 alias | v0.8 spec |
|---|---|
| `bg-brand-primary` (+ `-hover/-subtle`) | `bg-accent-brand-{normal,strong,bg,bgHover}` |
| `bg-brand-secondary` (+ `-hover/-subtle`) | `bg-ai-{normal,strong,hover}` (NOVA Purple) |
| `bg-primary-normal/-strong` (rc.0 alias) | `bg-accent-brand-{normal,strong}` |

> `accentAction.normal/strong` (Black 버튼) 는 v0.7부터 신규 토큰이며 v0.8에서 변동 없음.

#### 상태 (status → state, danger → error)

| v0.7 alias | v0.8 spec |
|---|---|
| `bg-status-success` (+ `-hover`) | `bg-state-success` (hover variant 흡수) |
| `bg-status-warning` (+ `-hover`) | `bg-state-warning` |
| `bg-status-danger` (+ `-hover`) | **`bg-state-error`** (이름 변경 — `error`가 표준) |
| `bg-status-info` (+ `-hover`) | `bg-state-info` |

#### Typography

| v0.7 alias | v0.8 spec |
|---|---|
| `text-polaris-display-lg` | `text-polaris-display` |
| `text-polaris-display-md` | `text-polaris-title` |
| `text-polaris-heading-lg` | `text-polaris-heading2` |
| `text-polaris-heading-md` | `text-polaris-heading3` |
| `text-polaris-heading-sm` | `text-polaris-heading4` |
| `text-polaris-h1` ~ `-h5` | `-display` / `-title` / `-heading1` / `-heading2` / `-heading4` |
| `text-polaris-body` (+ `-lg/-sm`) | `-body1` / `-body1` / `-body2` |
| `text-polaris-meta` | `text-polaris-caption1` |
| `text-polaris-tiny` | `text-polaris-caption2` |
| `text-polaris-caption` (suffix 없음) | `text-polaris-caption1` |

#### Radius / 컬러 ramp

| v0.7 alias | v0.8 spec |
|---|---|
| `rounded-polaris-full` | `rounded-polaris-pill` |
| `bg-blue-5` (+ green/orange/red/purple/sky-blue/blue-supplementary/violet/cyan/yellow/dark-blue) | `bg-blue-05` (선행 0 — 다른 step과 시각 정렬) |

> Gray ramp (10–90) 은 영향 없음. `-50` / `-500` 같은 더 큰 step도 negative-lookahead로 보호됩니다.

### 3.2 컴포넌트 BREAKING — props / 식별자

| 이전 (v0.7) | 이후 (v0.8) | 비고 |
|---|---|---|
| `<Button variant="outline">` | `<Button variant="tertiary">` | rc.0 시절 임시 이름. v0.7 spec("tertiary") 정렬 |
| `<Input hint="…">` | `<Input helperText="…">` | HTML spec / shadcn 컨벤션 정렬 |
| `<Textarea hint="…">` | `<Textarea helperText="…">` | 위와 동일 |
| `<Switch hint="…">` | `<Switch helperText="…">` | 위와 동일 |
| `<Checkbox hint="…">` | `<Checkbox helperText="…">` | 위와 동일 |
| `<FileInput hint="…">` / `<FileDropZone hint="…">` | `helperText="…"` | 위와 동일 |
| `<DateTimeInput hint="…">` / `<TimeInput hint="…">` | `helperText="…"` | 위와 동일 |
| `<Progress tone="success">` | `<Progress variant="success">` | `variant`로 통일 (Badge·Button·Alert 패턴) |
| `<Stat deltaTone="positive">` | `<Stat deltaVariant="positive">` | 한 컴포넌트 안에서도 `deltaTone` vs `tone` 혼재 해결 |
| `<HStack gap="md">` | `<Stack direction="row" gap="md">` | 단일 `<Stack>`로 통합. 임포트도 `Stack`만 |
| `<VStack gap="md">` | `<Stack gap="md">` | column 이 `<Stack>` 기본값 |

#### 포커스 링 통일

Button의 `ai` variant가 별도 `focus-visible:outline-ai-normal`을 사용하던 일관성 결함을 수정 — **모든 variant 공통 `focus-visible:shadow-polaris-focus`** (3px 시스템 포커스 링, light/dark 자동). Calendar / Drawer / Pagination / Ribbon 등도 동일 토큰으로 일원화.

> 컨슈머 코드에서 `focus-visible:outline-*` 을 직접 박았던 곳이 있다면 `focus-visible:shadow-polaris-focus` 로 옮겨 일관성을 유지하세요.

### 3.3 컴포넌트 enhancements — 디자인팀 컨펌 후 조정 가능

추천안으로 적용된 항목 — API 자체는 유지하지만 디자인팀 컨펌 후 토큰 수치만 미세 조정될 수 있습니다.

#### `Checkbox variant="ai"` (신규)

```tsx
<Checkbox variant="ai" label="AI 추천 적용" />
<Checkbox variant="ai" label="Polaris GPT-4 사용" defaultChecked />
```

- 기본 `default` (Brand Blue) — 일반 폼 / 동의 / 옵션 선택용. v0.7 동작 그대로.
- `ai` (NOVA Purple) — `<Button variant="ai">` / `<Badge tone="ai">` 와 시각 페어. AI 기능 명시 surface에서 Brand Blue로 위장하지 않게.

#### `DialogFooter fullWidthButtons`

```tsx
<DialogFooter fullWidthButtons>
  <Button variant="tertiary">취소</Button>
  <Button>저장</Button>
</DialogFooter>
```

- `false` (default): 오른쪽 정렬 + 자연 폭 (v0.7 동작 그대로)
- `true`: 직접 자식 `<button>` 을 `flex-1`로 늘려 row 폭을 균등 분할. 모바일 narrow modal / drawer-style 시트에서 두 액션이 half-width 페어로 읽힐 때.

> Anchor 액션(`asChild` + `<a>`)이 row에 섞여 있다면 그 element에 `className="flex-1"` 을 직접 붙이세요. `[&>button]:flex-1` 셀렉터가 button만 잡습니다.

## 5. 알려진 caveat

- **codemod 의 JSX 패턴은 opening-tag가 `>` 없이 한 줄에 들어와야** 정확히 매칭. 멀티라인 prop spread / 속성값에 `>` 포함된 경우 수동 review 권장. 잔여 패턴은 `grep -rn "hint=\|deltaTone=\|variant=['\"]outline\|<\(H\|V\)Stack" src/` 로 한 번 더 확인.
- ~~HStack/VStack import 라인에 이미 `Stack` 이 import 되어 있으면 codemod 결과가 중복 식별자를 만들 수 있습니다. TypeScript가 잡으니 한 줄에 합치세요.~~ → **rc.6에서 import dedupe pass 도입으로 자동 처리됨**. 이미 `Stack`을 import한 파일에 `HStack` / `VStack`이 함께 있어도 codemod가 binding 기준으로 중복을 정리해 단일 `Stack` 항목만 남깁니다.
- **`HStack as Row` 같은 alias된 import는 codemod가 abort합니다** (rc.7 신규 동작). `<Row>` JSX 태그가 codemod의 `<HStack>` / `<VStack>` 매칭 패턴에 안 잡혀서 `direction="row"` 속성이 누락되는 시각 회귀를 silent하게 만들기 때문에, 그런 파일은 codemod가 건드리지 않고 stderr에 명시적 안내를 출력합니다. 해결: alias를 풀어 `<HStack>` / `<VStack>` 그대로 쓰도록 import를 정리한 뒤 codemod 재실행.
- **rewrite destination namespace가 로컬 식별자와 충돌하면 codemod가 abort합니다** (rc.7 신규 동작). 예: 파일에 `const line = { … }` 가 있고 `surface.border` (rewrite 시 `line.neutral`) 가 같이 있으면, rewrite를 그대로 적용했을 때 `line.neutral` 이 *로컬 line*에 binding되는 silent semantic bug가 발생. codemod가 충돌을 사전 감지해 파일을 건드리지 않고 명시적 안내 출력. 해결: 로컬 식별자 rename 후 codemod 재실행. 같은 패턴이 `import { ai as aiToken }` 같은 alias된 polaris import에도 적용 (`ai` 바인딩이 다른 alias로 묶여 있어서 codemod가 추가하려는 `ai`와 충돌).
- **package.json의 dependency 범위**: `^0.7.x` 를 그대로 두면 v0.8을 못 받습니다. 명시적으로 `^0.8.0` 으로 올리세요.
- v0.6 이전(`fg-*`/`surface-*`)에서 한 번에 v0.8로 점프하는 경우 `polaris-codemod-v08` 한 번이면 충분합니다 — codemod table에 v0.6 alias가 모두 들어 있어요. v0.7 codemod를 먼저 돌릴 필요 없음.
- **`@polaris/ui` 자체 소스를 codemod target에 포함하지 마세요** — `packages/ui/src/{tokens,styles,tailwind}` 에는 마이그레이션 안내용 docstring에 옛 alias 이름이 의도적으로 들어 있어 false-positive 가 큽니다.

### Conflict abort — 무엇을 보고 어떻게 풀까

codemod가 conflict를 감지하면 stderr에 다음과 같이 출력하고 exit code 1로 끝납니다:

```
⚠️  2 files skipped due to namespace conflicts. Resolve manually before re-running:

  src/lib/colors.ts
    - would rewrite to 'line.*' but file declares a local 'line' (const/let/var/function/class). Codemod aborts to avoid binding rewritten code to your local — rename the local first.
  src/components/CardRow.tsx
    - imports HStack/VStack with an alias (e.g. `HStack as Row`). Codemod can't rewrite `<Row>` JSX tags to inject `direction="row"` — replace the alias with bare HStack/VStack first, or convert tags by hand.

Files above were left untouched. Other files were processed normally.
```

해결 패턴:
- **`local-binding`** — 로컬 식별자를 rename (예: `const line = …` → `const lineToken = …`). 또는 polaris token에 진짜 bind하려는 의도였다면 로컬을 지우면 됨.
- **`aliased-polaris-import`** — alias를 풀어 bare 이름으로 쓰거나, alias 사용처를 모두 다른 이름으로 옮긴 뒤 import 정리.
- **`aliased-stack-import`** — `import { HStack as Row }` → `import { HStack }` 로 바꾸고 모든 `<Row>` 를 `<HStack>` 으로 일괄 치환 (codemod가 그 후 재실행 시 `<HStack>` → `<Stack direction="row">` 까지 처리).
